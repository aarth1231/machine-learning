# Machine Learning Practice & Portfolio

Welcome to my Machine Learning repository! This repository tracks my hands-on journey learning data science, preprocessing techniques, and implementing foundational machine learning algorithms using **Python** and **Jupyter Notebooks**.

---

## ЁЯУБ Repository Structure & Content

The project files are categorized below by their corresponding machine learning subfields, algorithmic tasks, and dataset dependencies:

### 1. Regression Analysis
*   `homeprices.csv` & `hiring.csv` тАФ Benchmark datasets used for numeric trend forecasting.
*   `exc_multi_lin_reg_hiring.ipynb` тАФ Multi-variable linear regression implementation.

### 2. Classification Workflows
*   `binary_clas_log_reg.ipynb` & `single_classi_logistic_reg.ipynb` тАФ Fundamental binary and single-class classification using Logistic Regression.
*   `exc_multi_classi_log__reg.ipynb` тАФ Multi-class extensions for handling non-binary categorical targets.
*   `knn.ipynb` тАФ Instance-based classification using the K-Nearest Neighbors architecture.
*   `svm.ipynb` & `exc_svm.ipynb` тАФ Support Vector Machines with targeted margin optimization parameters.
*   `email_spam_detaction.ipynb` тАФ Natural language processing pipeline for spam text filtering.

### 3. Tree-Based Models & Ensemble Architecture
*   `tree.ipynb` & `exc_tree.ipynb` тАФ Core implementations of conditional Decision Trees.
*   `bagging.ipynb` тАФ Bootstrap Aggregation ensemble modeling designed to curb variance.
*   `exc_forest.ipynb` тАФ Random Forest assembly targeting high-dimensional tabular datasets.
*   `gradinet.ipynb` & `exc_gradient.ipynb` тАФ Sequential optimization pipelines built via Gradient Boosting methods.

### 4. Dimensionality Reduction & Tuning
*   `clustering.ipynb` тАФ Unsupervised partition sorting routines.
*   `pca.ipynb` тАФ Feature space optimization using Principal Component Analysis.
*   `hyper_parameter.ipynb` тАФ Operational tuning framework deploying parameter search optimization.
*   `L1_L2.ipynb` тАФ Cost function regularization strategies comparing Ridge and Lasso penalties.

### 5. Feature Engineering & Domain Datasets
*   `one_hot_encode.ipynb`, `one_hot_encode.ipynb` & `exc_one_hot_encoding.ipynb` тАФ Preprocessing workflows resolving structural categorical variables.
*   **Structured Medical Profiles:** `heart.csv`, `heart_disease.csv`, `heart_disease.ipynb`, `NutriRisk AI.ipynb`, and `diabetes_prediction_india (1).csv`.
*   **Operational & Demographics Benchmarks:** `carprices.csv`, `HR_comma_sep.csv`, `insurance_data.csv`, `Melbourne_housing.csv`, `moblie_classifier.csv`, `salaries.csv`, `spam.csv`, `Student_Performance_Factors.csv`, `test_scores.csv`, `titanic.csv`, `twst_train.ipynb`, and `WA_Fn-UseC_-Telco-Customer-Churn.csv`.
